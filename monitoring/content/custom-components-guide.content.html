Custom Components Guide

Create your own template

By defining a component tag, you can template data variables into the page, for example this template:

  
    
    {{my-data-greeting}} {{my-data-name}}

Turns into:

{{my-data-greeting}} {{my-data-name}}

Now, this isn't much use on its own - you may as well just write those values out into the right places.

Coming soon: proper templates that will can be defined once, and then reused throughout the page.

Data Aware Component with data-source-url

Here is an example of a data-aware component, that populates itself from the supplied data-source-url:

  
    
    
      
  • Status code: {{statusCode}}
    • 
      
  • Message: {{message}}
    •

Turns into:

  • Status code: {{statusCode}}
  • Message: {{message}}

Works great! Data Aware Components should be super easy to connect up data from a server, and render it how and where you want it, without arduous templating.

Accessing Hidden Data

Behind the scenes, the component is going off and making AJAX calls using jQuery to load the data. If there's a problem, you can find out through dataSource variables locked on to the object.

If for example, you change the template to include these values:

  
    
    
      
  • {{dataSourceStatus}}
    • 
      
  • {{dataSourceUrl}}
    •

Then we get lots of information:

{{dataSourceStatus}}
{{dataSourceUrl}}
{{dataSourceData}}

Data Aware Component with data-source-template

Here's the same component as above, but this time using a data-source-template to create its own data-source-url:

  
    
    
      
  • Status code: {{statusCode}}
    • 
      
  • Message: {{message}}
    • 
    

    
{{dataSourceData}}

Turns into:

  • Status code: {{statusCode}}
  • Message: {{message}}
{{dataSourceData}}

Now you're thinking in templates! Soon you'll be able to spin this out with custom templates to quickly build data driven components that are styled how you want them to be.

Where this really kicks in is when we can define data-driven templates using templates, and then you're just plugging in the minimum amount of information to grab information from the server. That's how Status Checkers work behind the scenes.
