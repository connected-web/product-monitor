Custom API Endpoint Guide

Create your own endpoints

Using the template below, you can create and register your own endpoints with this server by just making files in the /monitoring/api/ directory.

For endpoints to be compatable with the server, they must meet the following criteria:

, i.e. module.exports = {} , i.e. instance.route = "/api/my-route"; - this defines the endpoint that your script will run on. , i.e. instance.cacheTime = "10 minutes" - by default 1 hour caching is enabled on all new PI endpoints, you can change this value by passing in [length] [unit] as a string, for example "30 seconds", "5 minutes", "2 hours", etc.. , i.e. instance.configure = function(config) {...} - this allows config values to be passed in from the server instance, such as secure keys or passwords. , i.e. instance.render = function(req, res) {...} - this is where you get to apply custom logic, based on a request, and form an appropriate response.

This information is passed through to an Express JS server instance, and is registered with the server using following steps:

instance.configure(applicationConfig);
expressServer.get(instance.route, instance.render);

Example template

This is an example of a working API entry:

var instance = function() {}

var server = false;

instance.route = '/api/exampleEndpoint/:name';

instance.configure = function(config) {
  server = config.server;
}

instance.render = function(req, res) {
  var data = {};

  // read parameter from route
  var name = req.params.name || false;

  // form response
  var data = {
    suppliedName: name
  };

  // send response
  res.jsonp(data);
}

module.exports = instance;

The above example works by serving up a custom endpoint on the URL /api/exampleEndpoint/:name, where :name is a dynamic parameter understood by Express JS's routing system.

The script takes the :name and returns a data object in the form:

{
  "suppliedName": ":name"
}

This data can then be wired into a custom component template, for more details on this see the Custom Components Guide.
